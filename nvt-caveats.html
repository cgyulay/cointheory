NVT Ratio Caveats

Though the network value to transaction ratio (NVT) is a useful measure for indicating the relative value of a cryptoasset, calculating the exact transaction volume is difficult because of transactional change. On the blockchain, there are no balances associated with any address or user. Instead, there is only a history of transactions. If user A wanted to send user B some Bitcoin, for example, the input to the transaction would be one of (or some series) of Bitcoin amounts from previously received transactions, not the exact amount. Thus, there will be some change to return to the initial sender in the transaction. If the change is returned to the sender's address, then it is straightforward to separate the payment from the change and to determine the transaction volume. However, if two new addresses are used for destination and change, it is not possible to determine transaction volume with 100% certainty. An example of such a transaction can be seen on the Bitcoin blockchain here. We and others like Blockchain.info are developing heuristics to tighten these estimations. This is an ongoing open question and area of research.

As of right now, we do not employ any heuristics on our data to estimate payment vs. change. We only remove the change if the change address is 100% knowable from the transaction details. This means our NVT overestimates transaction volume and provides a lower bound on true underlying NVT. This may at first seem like a problem, but because these behaviors are consistent over time, NVT still functions as a useful relative indicator. As stated, our goals are to better estimate transaction volume and to better understand change patterns across currencies so cross-currency NVT comparison becomes more viable.
